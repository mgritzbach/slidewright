# Complaint-derived reliability contract

Public complaints about Codex presentation work establish these mandatory behaviors:

1. Preflight the presentation runtime and report one exact remediation when unavailable. Never silently install or switch tools.
2. Never claim a PPTX was created until the file exists, has nonzero size, passes ZIP/OOXML checks, and the delivered path resolves.
3. Keep visible text and semantic slide content native and editable. Do not substitute HTML, PDF, or full-slide images unless the user explicitly requests them.
4. For release-quality output, open the final fixture in Microsoft PowerPoint without a repair dialog and retain the before/after object manifest.
5. Emit per-slide previews so users are not dependent on Codex's Office preview surface.
6. Treat existing templates, masters, placeholders, themes, and fonts as preservation constraints. If import is unavailable, say so before generation.
7. Apply small edits to stable named objects. Recompile, audit, render, and compare; never globally restyle a deck to change one object.
8. Use robust defaults so ordinary prompts cannot bypass the formatting workflow. Reject conflicting instructions with a concise explanation.
9. Publish exact quality evidence and remaining deviations. Do not use “pixel-perfect,” “template-preserving,” or “cross-platform” without the named proof.

Until a contract item is implemented, surface it as a limitation rather than improvising a weaker substitute.
