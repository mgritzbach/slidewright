# Slidewright 0.4.0

Slidewright 0.4.0 is the current installable release of the formatting-safe, native-editable PowerPoint compiler for Codex. It publishes the product and evidence hardening added after v0.3.0 without overstating the three human- or host-dependent complaint goals that remain open.

## What is new

- Source-native composition can build a new deck from a licensed or explicitly authorized reference presentation while retaining native master, layout, theme, guide, logo, limiting-line, and recurring-chrome relationships.
- The consulting layout grammar now includes true regular triangle-through-dodecagon systems, protected triangle text placement, adjacent no-leader labels, quadrant synthesis, intrinsic chevron flows, and honeycomb, pyramid, and square icon networks.
- Required network connectors render beneath nodes and text, terminate invisibly under their endpoints, and match the destination rim color and weight.
- A semantic single-focus contract highlights one peer or one synthesis outcome without changing peer geometry or typography.
- Authenticated Google Slides, Canva, PowerPoint macOS, and Keynote interoperability runners now produce credential-free, artifact-bound evidence instead of capability-detection claims.
- C02 primary-client and C13 blind-human-study workflows are fail-closed, privacy-preserving, and reproducible.

## Verified release state

- All 36 product goals are credited.
- 19 of 22 complaint-derived goals are credited.
- Generated text, shapes, tables, charts, and diagrams remain native and editable unless an interoperability matrix explicitly reports an application conversion as changed or unsupported.
- Demo and benchmark pipelines treat warnings as failures and require compile, lint, render, OOXML audit, and full-size review evidence.

## Deliberately open evidence gates

- C02 still requires fresh primary Codex Desktop and VS Code tasks against this exact public release.
- C13 still requires one independent blind expert and five independent target users to complete the prepared 21-design study.
- C19 still requires native PowerPoint macOS and Keynote runs, the final authenticated Canva cleanup-bound capture, and a single published six-application matrix on one exact source and commit.

These are not missing implementation claims. They are intentionally external acceptance boundaries and remain `0` until their named evidence exists.

## Upgrade

Refresh the Slidewright plugin through the Codex plugin installer, then start a new Codex task so the client loads the v0.4.0 registry. Confirm that the installed manifest and `SKILL.md` come from the same release before running the preflight.
