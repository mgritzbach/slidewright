# Build Week and GitHub-star plan

## Product thesis

**Visual reference → editable PowerPoint → forensic proof that formatting survived.**

The moat is not another slide generator. It is a formatting compiler and evidence pipeline: native text, deterministic geometry, conventional typography, real groups, rendering, and machine-readable audits.

Current proof boundary: the six-fixture suite validates deterministic export from a shared design manifest; one independent opaque-image fixture validates anti-circular reconstruction; and one MIT golden template validates exact named-placeholder copy edits. These narrow proofs do not establish broad multi-domain image understanding or general structural PPTX import.

## Release blockers

1. Expand the public benchmark from 6 to at least 20 licensed or owned fixtures.
2. Expand independent image ingestion beyond the current one-fixture proof.
3. Expand copy mutation beyond the current -25% and +25% English fixtures to translation and dense-copy relayout.
4. Expand missing-font proof to licensed embedding and cross-platform substitution behavior.
5. Expand the exact named-placeholder golden-template path to complex licensed decks and structural edits.
6. Exercise compile, lint, render, audit, and fidelity checks in CI on supported platforms.
7. Record a 2:30 demo: reference → generation → ungroup/edit → rerender → scorecard.
8. Run five target-user tests and measure cleanup minutes and first-open acceptance.
9. Package the judge submission: repository access, public video, screenshots, test instructions, description, and verified feedback session ID.
10. Record ownership/license provenance for every fixture, font, icon, and dependency.

## Complaint-driven build order

The public complaint ledger changes the implementation order:

1. **Trust the file:** C03–C06. Runtime preflight, repair-free OOXML, verified delivery, and preview-independent handoff.
2. **Trust the editability:** C07–C11. Complete native coverage, no text rasterization, templates, masters, fonts, and advanced formatting.
3. **Trust the workflow:** C12–C16. Prompt robustness, professional review, defect detection, mutation resilience, and surgical iteration.
4. **Expand the surface:** C17–C21. Existing-deck import, native charts/tables/diagrams, interoperability, transparent dependencies, and optional PowerPoint control.
5. **Prove it publicly:** C22. A committed, versioned, fresh-machine-reproducible benchmark and scorecard.

## GitHub-star multipliers

- Immediate before/after gallery and a short animated demo in the README.
- Copy-paste quick start that succeeds on a fresh machine in under five minutes.
- Native tables, charts, connectors, images, nested groups, and semantic components.
- Three tutorials: recreate a design, generate from an outline, repair an existing deck.
- Honest platform matrix and limitations page.
- Versioned releases, changelog, lockfile, compatibility policy, and curated fixtures.
- Contribution guide, code of conduct, security policy, issue templates, and good-first-issue roadmap.
- Public benchmark leaderboard with community-contributed fixtures.

## Claims policy

Do not claim pixel-perfect recreation, font embedding, template preservation, cross-platform rendering, or universal group/ungroup behavior until the named proof exists. Publish exact metrics and remaining deviations.
