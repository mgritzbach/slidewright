# OpenAI Build Week submission requirements

Verified on **July 14, 2026** against the [official OpenAI Build Week page](https://openai.com/build-week/) and [Devpost challenge page](https://openai.devpost.com/).

## Chosen category

**Work & Productivity** — Slidewright makes teams faster by removing formatting repair from a common knowledge-work workflow.

## Key dates

- Submission deadline: **Tuesday, July 21, 2026 at 5:00 PM Pacific**.
- Judging: July 22 through August 5, 2026.
- Winners announced: on or around August 12, 2026.
- Free-credit request deadline: July 17, 2026 at 12:00 PM Pacific, while supplies last.

## Mandatory build and submission items

- A working project built with Codex and GPT-5.6.
- Category selection.
- Project description explaining features and functionality.
- Public YouTube demo under three minutes, with audio showing the project and explaining use of Codex and GPT-5.6.
- Repository URL. It may be public with relevant licensing, or private and shared with `testing@devpost.com` and `build-week-event@openai.com`.
- README with setup instructions, sample data if needed, and clear run instructions.
- README explanation of how Codex accelerated the work, where human product/engineering/design decisions were made, and how GPT-5.6 and Codex contributed.
- `/feedback` Codex Session ID for the thread where most core functionality was built.
- Because this is a plugin/developer tool: installation instructions, supported platforms, and a judge path that does not require rebuilding from scratch.

## Judging criteria

The four Stage Two criteria are equally weighted:

1. **Technological implementation:** thorough, skillful Codex use and a working non-trivial implementation.
2. **Design:** a complete and coherent runnable experience, not only a technical proof of concept.
3. **Potential impact:** a credible specific problem for a real audience, demonstrably addressed.
4. **Quality of the idea:** creativity, novelty, and genuine problem-space understanding.

## Rules that affect this repository

- New projects are allowed. Existing projects must be meaningfully extended during the submission period, with dated Codex sessions or commit history separating new work.
- The project must install and run consistently as shown in the demo.
- Third-party SDKs, APIs, assets, and data require proper authorization and license compliance.
- The submission must be original and owned by the entrant; compliant open-source components are permitted.
- Judge access must remain free and unrestricted through the judging period.
- Submission materials must be in English or include an English translation.
- After the deadline, the submitted entry cannot normally be substantively changed.
