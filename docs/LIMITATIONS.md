# Current limitations

- C12's guarded request path proves data/control separation for the committed strict request schema and adversarial corpus. It does not prove that every natural-language conflict in every language will be classified. Safety does not depend on phrase recognition: prompt text cannot control commands, paths, stages, thresholds, receipts, or publication.
- The current guarded request schema is intentionally narrower than all developer primitives: it supports the proven 1280x720 compiler layouts, a symmetric margin floor, no image-bearing request specs, and zero-picture plan-bound audits. Broader canvases and true visual-asset allowlists require their own evidence before joining this qualified path.

- The semantic compiler currently supports a narrow set of layouts. Independent image ingestion is proven on one original invitation fixture, not yet a broad multi-domain benchmark; pixel-only font identity remains an explicit guess.
- Exact named-placeholder editing remains the narrowest supported source-deck mutation path. The credited C10 matrix extends preservation evidence to four licensed template families and 39 slides containing native charts/workbooks, tables, images, repeated layouts, multiple masters/layouts, placeholders, themes, notes, hyperlinks, media, spacing, and recurring chrome through two PowerPoint serialization cycles. C17 separately proves lossless structural import/export on four licensed decks for hierarchy, text runs, tables, charts, diagrams, notes, and semantic reading order. That importer preserves native OOXML but does not promise unrestricted edits or semantic reconstruction of every third-party Office object or SmartArt program.
- Exact visual rendering depends on fonts being present or validly embedded. Missing requested families block with an explicit fallback suggestion. C11 proves that one OFL-licensed four-style embedded fixture retains its native run formatting through two PowerPoint cycles; Slidewright does not automatically embed, license, or redistribute arbitrary user fonts.
- Native grouping requires deterministic OOXML normalization after artifact export.
- PowerPoint group round-trip testing currently runs only on Windows with Microsoft PowerPoint installed.
- The direct PowerPoint adapter is an optional Windows-only fallback proven for one named native text/bold edit inside one native group. Stable fine-grained iteration is separately bounded to Slidewright plans with unique named objects and the six `c16-v1` edit contracts; arbitrary selection flows and arbitrary layout restructuring remain unsupported.
- Similarity scores include renderer and font-antialiasing differences and must be read beside the structural audit and full-size review.
- Automatic continuation splitting is bounded to the proven hero, two-column, section-detail, and continuation layouts. The pinned C15 corpus covers -25%, +25%, one human-authored German translation, and up to 4x flexible copy in Arial; Slidewright does not translate language, claim arbitrary scripts/layouts, or permit unbounded slide growth. Content that cannot fit the bounded layouts or exceeds the configured slide ceiling fails before export with actionable diagnostics. Broad accessibility checks remain a roadmap goal. Native Office horizontal/vertical charts and tables are proven in the controlled C08 surface, and C18 qualifies five exact real-PowerPoint edits with nine controls and 24 reviewed renders. C17 preserves declared imported surfaces losslessly but does not qualify arbitrary chart/table/diagram creation, mutation, or readability.
- C19 remains uncredited until one identical source deck has independently passed native-edit and re-export evidence in PowerPoint Windows/macOS, Google Slides, Keynote, LibreOffice, and Canva. Detecting an installed application or successfully opening a file is insufficient.
- Automated plan and rendered-layout checks cover known canvas/parent clipping, overflow, overlap, contrast, declared alignment, wrapping, crowding, and chart-readability constraints. They complement, but do not replace, full-size review or prove professional quality across arbitrary designs.
- The feedback-contract benchmark proves absolute text separation, exact no-split/center/two-thirds headline intervals, growing title bands, ordered topic ownership, and inherited empty-bullet cleanup on one synthetic 34-slide outline plus one PowerPoint-authored two-slide source template. It does not yet establish those guarantees across arbitrary third-party masters, SmartArt, embedded media, or every PowerPoint placeholder inheritance variant.
